# COSMIC‑ARC Prototype

**COSMIC‑ARC** (Cosmic Accelerator Research & Compliance) is a minimal, end‑to‑end prototype that showcases the core research workflow required for the DARPA SN‑26‑56 concept. It demonstrates the full **golden‑path**:

1. **Ingestion** – raw events from a mock runtime adapter are normalised into a universal `EventEnvelope`.
2. **Timeline reconstruction** – `TimelineEngine` orders events by logical clock, building a parallel‑state view.
3. **Two‑stage anomaly detection** –
   - *Feature extraction & signal enrichment* (`featureExtraction.ts`).
   - *Scoring, ranking & triage* (`scoring.ts`).
4. **Evidence packaging** – `EvidencePack` emits a manifest, findings, artifact references, integrity hashes, and a replay‑ready timeline snapshot.
5. **Replay harness** – (stubbed in `packages/replay`) can re‑hydrate a session and inject anomaly scenarios for deterministic demos.

The prototype is deliberately **vendor‑agnostic** – adapters are abstract interfaces (`@cosmic-arc/adapters`) that can be implemented for any accelerator (GPU, TPU, NPU, etc.).

```
cosmic-arc-prototype/
├─ apps/
│  ├─ demo/          # Minimal demo app (ts-node) – runs the full workflow
│  ├─ console/       # Placeholder for a future UI
│  └─ api/           # Placeholder for a future Fastify API
├─ packages/
│  ├─ schema/        # Zod schemas & EventEnvelope definition
│  ├─ adapters/      # Adapter interfaces (runtime, host, trace, synthetic)
│  ├─ ingestion/     # IngestionPipeline – normalises raw streams
│  ├─ timeline/      # TimelineEngine – ordered parallel‑state timeline
│  ├─ graph/         # CorrelationGraphService – cross‑layer relationship store
│  ├─ anomaly/       # Feature extraction + scoring (two‑stage pipeline)
│  ├─ evidence/      # EvidencePack definition & builder helper
│  ├─ replay/        # ReplayRunner – deterministic session re‑hydration
│  └─ … (future)    # Additional services (e.g., COSMIC engine bindings)
├─ tsconfig.json
└─ package.json (workspace root)
```

## Getting started
```bash
# Clone the repo (already done if you followed the earlier steps)
git clone https://github.com/sokpyeon/cosmic-arc-prototype.git
cd cosmic-arc-prototype

# Install all workspace dependencies
npm install

# Verify TypeScript compiles cleanly
npx tsc --noEmit
```

### Run the demo (full golden‑path)
```bash
cd apps/demo
npm run start   # or: npx ts-node src/demo.ts
```
You should see a JSON dump of an `EvidencePack` similar to:
```json
{
  "manifest": {
    "title": "Demo Evidence Pack",
    "generatedAt": "2026-03-27T18:17:01.260Z",
    "sessionId": "unknown",
    "traceId": "unknown"
  },
  "findings": [],
  "artifacts": [],
  "integrity": {},
  "replay": { "timelineSnapshot": [] }
}
```
The demo currently uses a synthetic runtime adapter that emits a few mock kernel‑launch events, runs them through the pipeline, and produces the evidence pack.

## Extending the prototype
- **Implement real adapters** in `packages/adapters` (e.g., a CUDA driver adapter, host‑API interposition, trace‑file import, or synthetic test generator).
- **Wire the correlation graph**: feed events into `CorrelationGraphService` to build cross‑layer relationships (`caused_by`, `reads_from`, etc.).
- **Enrich the anomaly pipeline** with richer features (memory‑transfer irregularities, sync‑queue anomalies, etc.) and a proper scoring model.
- **Populate the evidence pack** with actual artifact files, SHA‑256 hashes, and a populated `timelineSnapshot`.
- **Build a UI** in `apps/console` or an API in `apps/api` to visualise the timeline, graph, and findings.
